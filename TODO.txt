✅ PHASE 1 — Data Cleaning & Restructuring
1. Load your raw data

Your CSV with the groupings

Your Stimuli.xlsx with item metadata

2. Convert group strings → one sound per row

Use your explode_groups() function to get:

| PARTICIPANT | SLIDE | GROUP # | SOUND_ID |

This is now “tidy” and analysable.

3. Ensure data integrity

Check:

No empty SOUND rows

No non-numeric IDs

No missing participants

SLIDE values = 1 and 2

Each trial has 20 unique sounds

Each participant has both trials

Group sizes make sense (e.g., no 0-size groups)

4. Join stimulus metadata

Using SOUND_ID, merge with your Stimuli.xlsx so each sound has:

its vowel

acoustic measures (F1/F2/etc., if present)

any category/label structure you need

Now you have:

| PARTICIPANT | SLIDE | GROUP # | SOUND_ID | vowel | F1 | F2 | … |

✅ PHASE 2 — Descriptive Behaviour & QC
5. Calculate cluster size statistics

For each participant × trial:

how many clusters did they produce?

what were the cluster sizes?

do some participants cluster very differently?

Plot distributions to catch outliers.

6. Coverage check

Make sure each participant used all 20 sounds per trial:

count distinct SOUND_ID per participant × trial

flag participants with missing sounds

7. Stimulus-level sanity

Confirm your stimuli sheet correctly maps:

sounds 1–20 → trial 1

sounds 21–40 → trial 2
(or whatever your scheme is)

✅ PHASE 3 — Build the Core Similarity Representation

This is the heart of the analysis.

8. Generate pairwise co-occurrence data

For each similarity group, create all combinations:

If group = {4, 7, 20} → generate pairs:

(4,7), (4,20), (7,20)

Do this per:

participant

trial

group

9. Aggregate across participants

For each trial separately:

count how many participants placed each pair (i,j) together

produce a 20×20 similarity matrix per trial

Values represent:

“How often sounds i and j were grouped together.”

10. Normalize (optional)

Compute:

proportions

z-scores

normalized similarity

Optional but often needed for MDS or clustering.

✅ PHASE 4 — Visualization & Structural Analysis
11. Heatmaps

Plot the similarity matrix as a heatmap:

reveals cluster patterns

lets you spot strong vs weak similarities

12. Multidimensional Scaling (MDS)

Take the similarity matrix → convert to a distance matrix → run classical MDS.

You’ll get:

a 2D perceptual map

where proximity = similarity

13. Hierarchical Clustering (Agglomerative)

Build dendrograms to see hierarchical grouping structure.

14. Compare to phonetic features

Overlay:

vowel quality

F1/F2 position

phonological class

speaker identity

Ask:

Do perceptual clusters align with phonetic reality?

Are some vowels perceptually merged?

✅ PHASE 5 — Statistical Questions

Now that the perceptual structure exists, you can ask:

15. Within-trial structure

which sound pairs are most/least similar?

are clusters small or large overall?

16. Across-participant consistency

measure agreement using consistency indices

test which pairs consistently cluster together

17. Effects of stimulus properties

Using metadata:

Do same-vowel pairs cluster together more often?

Does F1/F2 Euclidean distance predict similarity?

Are central vowels treated inconsistently?

18. Group comparisons (if applicable)

If participants have groups (L1 vs L2, etc.):

compare similarity matrices

compare MDS spaces

test differences in specific pair co-occurrences

🎯 PHASE 6 — Reporting & Interpretation
19. Summarize

Describe participant clustering behaviour

Describe perceptual structure

Compare it to phonetic structure

Highlight strong effects (e.g., frontness, vowel height)

20. Figures

Include:

2 MDS plots (one per trial)

2 heatmaps

dendrograms

cluster size distribution plots

pairwise similarity distributions

21. Interpretation

Explain:

which perceptual dimensions participants used

whether categories are crisp or fuzzy

how grouping relates to acoustic/phonological features
